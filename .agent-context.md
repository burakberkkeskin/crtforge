# 🤖 Agent Context: crtforge

This file provides high-level context, architecture, and technical details about the `crtforge` repository to assist AI agents in understanding and contributing to the project.

## 📋 Project Overview
`crtforge` is a Go-based CLI tool designed to simplify the creation of full-chain self-signed SSL/TLS certificates. It automates the generation of a complete chain:
1. **Root CA Certificate** (Self-signed)
2. **Intermediate CA Certificate** (Signed by Root CA)
3. **Application/Leaf Certificate** (Signed by Intermediate CA)

The tool is designed for local development, home labs, and testing environments.

## 🛠 Tech Stack
- **Language:** [Go (Golang)](https://golang.org/)
- **CLI Framework:** [Cobra](https://github.com/spf13/cobra)
- **Logging:** [Logrus](https://github.com/sirupsen/logrus)
- **External Dependencies:** `openssl` (invoked via `os/exec` for CA operations)
- **Containerization:** Docker (Dockerfile and Dockerfile.production)
- **CI/CD:** GitHub Actions (Automated builds and releases via Git tags)

## 🔑 Core Logic & Implementation Details

### Certificate Generation Flow
The standard workflow follows this chain of command:
`main.go` $\rightarrow$ `cmd/root.go` $\rightarrow$ `cmd/services/*`

1.  **Root CA:** Handled by `rootCaService.go`. Uses `openssl` to create the root cert and key.
2.  **Intermediate CA:** Handled by `intermediateCaService.go`. Uses the Root CA to sign a new intermediate CA.
3.  **Application Cert:** Handled by `appCrtService.go`. Uses the Intermediate CA to sign the leaf certificate.

### Key Command Flags (Cobra)
- `--root-ca`, `-r`: Sets the name of the Root CA.
- `--intermediate-ca`, `-i`: Sets the name of the Intermediate CA.
- `--trust`, `-t`: Automatically adds the Root CA to the system trust store.
- `--pfx`, `-p`: Generates a `.pfx` (PKCS#12) file for the application certificate.
- `--renew`, `-R`: **(New)** Renews only the leaf certificate using existing CA files.

### Configuration & Storage
- **Default Config Dir:** `$HOME/.config/crtforge`
- **Structure under Config Dir:**
  - `/rootCA/` (Contains `rootCA.crt`, `rootCA.key`, `rootCA.cnf`)
  - `/{CA_NAME}/` (Intermediate CA directory)
  - `/{APP_NAME}/` (Application certificate files)

## 🚀 Release Process
Releases are automated via GitHub Actions.
1.  Create a release branch from `master`.
2.  Tag the commit with semantic versioning (e.g., `v1.1.0`).
3.  Push the tag to GitHub.
4.  The `build-release.yml` workflow triggers, builds binaries/images, and creates the GitHub Release.

## 🤖 Guidelines for Agents
- **Logic Location:** When asked to change certificate generation logic, look in `cmd/services/`.
- **Flag/Command Changes:** When asked to add CLI options, look in `cmd/root.go`.
- **Dependencies:** Always assume `openssl` is required on the host system for CA operations.
- **Code Style:** Follow standard Go idioms and use `logrus` for all logging.
